## ✨ 新增功能

### 1. 系统学科助手
- 新增7个固定的学科助手：数学、物理、化学、生物、英语、历史、地理
- 每个助手都有专业的学科背景和教学特点
- 系统助手不可删除，确保核心功能稳定可用

### 2. 专业教学提示词
- 每个学科助手都配备了专业的教学提示词
- 提示词定义助手为专业教师身份，而非AI
- 强调学科特点和教学方法
- 注重培养学生的学科思维和学习能力

### 3. 学科知识库
- 每个学科助手自动关联对应的学科知识库
- 知识库与助手一一对应，便于管理学科资料
- 系统知识库不可修改关联关系，保证数据一致性

### 4. 助手保护机制
- 系统助手的提示词不可修改（前端显示为只读）
- 系统助手的知识库关联不可修改
- 系统助手不可删除
- 用户可以查看系统助手的提示词内容

### 5. 自动初始化
- 用户首次进入AI对话页面时自动创建系统助手
- 系统助手和知识库自动关联
- 无需手动配置，开箱即用

## 📚 学科助手详情

### 数学教学助手
- **学科**: 数学
- **特点**: 善于用通俗易懂的语言解释抽象的数学概念
- **教学方法**: 详细的解题步骤、思路分析、举一反三
- **培养目标**: 数学思维、解题能力、严谨的逻辑

### 物理教学助手
- **学科**: 物理
- **特点**: 将抽象的物理概念与生活实际相联系
- **教学方法**: 现象分析、原理讲解、图示说明
- **培养目标**: 科学思维、观察分析能力、实验能力

### 化学教学助手
- **学科**: 化学
- **特点**: 用实验现象和化学反应解释化学原理
- **教学方法**: 方程式书写、化学计算、对比归纳
- **培养目标**: 化学素养、实验能力、安全意识

### 生物教学助手
- **学科**: 生物
- **特点**: 用生动的例子解释生命现象和生物规律
- **教学方法**: 图表模型、系统讲解、联系实际
- **培养目标**: 科学探究能力、生态保护意识

### 英语教学助手
- **学科**: 英语
- **特点**: 注重听说读写全面发展，强调语言实际运用
- **教学方法**: 语法讲解、词汇扩展、情景对话
- **培养目标**: 语言运用能力、跨文化交际能力

### 历史教学助手
- **学科**: 历史
- **特点**: 讲述历史故事，让历史人物和事件生动起来
- **教学方法**: 史料分析、因果关系、时间轴梳理
- **培养目标**: 历史思维、批判性思维、家国情怀

### 地理教学助手
- **学科**: 地理
- **特点**: 用地图和图表展示地理信息和空间关系
- **教学方法**: 区域分析、案例研究、图文结合
- **培养目标**: 地理素养、读图析图能力、可持续发展观念

## � 使用说明

### 如何使用系统学科助手
1. 进入AI对话页面，系统会自动创建7个学科助手
2. 在助手列表中选择对应学科的助手（带"系统"标签）
3. 创建新对话，开始与学科助手交流
4. 学科助手会根据学科特点提供专业的教学辅导

### 系统助手自动创建逻辑

#### 创建时机
系统助手会在以下情况下自动创建：

1. **用户首次进入AI对话页面时**
   - 前端 `AIChat.vue` 的 `onMounted` 钩子会调用 `initSystemAssistants(userId)` 接口
   - 后端检查该用户是否已有系统助手，如果没有则自动创建

2. **换新数据库/新服务器时**
   - ✅ **会自动创建**
   - 当用户首次访问AI对话页面时，系统会检测到该用户没有系统助手
   - 自动为该用户创建7个学科助手和对应的知识库
   - 无需手动操作，完全自动化

#### 创建流程
```
用户进入AI对话页面
    ↓
前端调用 initSystemAssistants(userId)
    ↓
后端检查该用户是否已有系统助手
    ↓
如果没有 → 创建7个学科助手
    ↓
为每个助手创建对应的知识库
    ↓
建立助手与知识库的关联关系
    ↓
返回创建的助手列表
    ↓
前端重新加载助手列表并显示
```

#### 幂等性保证
- 系统助手的创建是**幂等**的
- 多次调用 `initSystemAssistants` 不会重复创建
- 后端通过 `userId + isSystem + subject` 组合判断是否已存在
- 如果已存在，直接返回现有的助手，不会创建新的

#### 数据库迁移场景

**场景1：全新数据库**
- 用户首次登录 → 进入AI对话页面 → 自动创建系统助手 ✅

**场景2：现有数据库升级**
- 现有用户进入AI对话页面 → 检测到没有系统助手 → 自动创建 ✅

**场景3：数据库备份恢复**
- 如果备份中包含系统助手数据 → 不会重复创建 ✅
- 如果备份中不包含系统助手数据 → 自动创建 ✅

**场景4：更换服务器**
- 新服务器 + 新数据库 → 用户访问时自动创建 ✅
- 新服务器 + 旧数据库迁移 → 根据数据库中是否存在决定是否创建 ✅

#### 技术实现细节

**后端逻辑** (`assistantController.js`)
```javascript
// 检查该学科的系统助手是否已存在
let assistant = await Assistant.findOne({ 
  userId, 
  isSystem: true, 
  subject: config.subject 
});

if (!assistant) {
  // 不存在才创建
  assistant = new Assistant({...});
  await assistant.save();
}
```

**前端调用** (`AIChat.vue`)
```javascript
onMounted(async () => {
  // 初始化系统学科助手
  await initSystemAssistants(userId.value);
  await loadAssistants(); // 重新加载助手列表
});
```

### 系统助手的特点
- 系统助手不可删除，确保始终可用
- 系统助手的提示词经过精心设计，不可修改
- 系统助手自动关联对应的学科知识库
- 用户可以查看系统助手的提示词内容
- 每个用户都有独立的系统助手实例（数据隔离）

### 系统助手 vs 用户助手
- **系统助手**: 固定的学科助手，提示词和知识库不可修改，不可删除，自动创建
- **用户助手**: 用户自定义的助手，可以自由编辑和删除，手动创建
