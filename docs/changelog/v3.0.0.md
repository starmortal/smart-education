## ✨ 新增功能

### 1. 实时通知推送
- 基于 Socket.IO 的实时通知系统
- 无需刷新页面即可接收最新通知
- 支持多种通知类型：注册欢迎、登录提醒、社区回复、点赞、关注、系统公告

### 2. 通知中心页面
- 全新的通知中心界面，统一管理所有通知
- 支持按状态筛选：全部、未读、已读
- 通知列表展示，包含时间、内容、操作按钮
- 一键标记已读/全部已读功能
- 点击通知可跳转到相关内容

### 3. 邮箱通知功能
- 重要通知支持邮件提醒
- 用户可自主开启/关闭邮箱通知
- 支持按通知类型设置是否接收邮件

### 4. 通知设置中心
- 个性化通知设置面板
- 邮箱通知总开关
- 各类型通知独立开关（注册、登录、回复、点赞、关注、系统公告）
- 设置实时保存

### 5. 侧边栏通知入口
- 侧边栏新增"通知中心"菜单项
- 实时显示未读通知数量（红点角标）
- 快速访问通知中心

### 6. 通知触发场景
- **注册欢迎**：用户注册成功时发送欢迎通知
- **登录提醒**：用户登录时发送登录通知
- **社区回复**：问题被回复时通知问题作者
- **点赞通知**：回答被点赞时通知回答作者
- **关注通知**：被其他用户关注时发送通知
- **系统公告**：管理员发布系统公告

## 📋 API 接口

### 通知相关接口
- `GET /api/notification/list` - 获取通知列表（支持分页和状态筛选）
- `GET /api/notification/unread-count` - 获取未读通知数量
- `PUT /api/notification/:id/read` - 标记单条通知为已读
- `PUT /api/notification/read-all` - 标记全部通知为已读
- `DELETE /api/notification/:id` - 删除单条通知
- `DELETE /api/notification/clear-all` - 清空所有通知

### 通知设置接口
- `GET /api/notification/settings` - 获取用户通知设置
- `PUT /api/notification/settings` - 更新用户通知设置

### Socket.IO 事件
- `notification:new` - 接收新通知
- `notification:read` - 通知已读状态更新
- `notification:delete` - 通知删除

## 📝 使用说明

### 如何使用通知中心
1. 点击侧边栏的"通知中心"菜单进入通知中心页面
2. 查看未读通知（红点角标显示未读数量）
3. 点击通知可查看详情或跳转到相关内容
4. 点击"标记已读"将通知标记为已读
5. 点击"全部已读"将所有通知标记为已读

### 如何设置通知
1. 进入通知中心页面
2. 点击右上角的"设置"按钮
3. 开启/关闭邮箱通知总开关
4. 根据需要开启/关闭各类型通知
5. 设置自动保存

### 邮箱通知说明
- 开启邮箱通知后，重要通知会同时发送到您的注册邮箱
- 支持的邮件通知类型：注册欢迎、社区回复、关注通知、系统公告
- 登录提醒和点赞通知默认不发送邮件
