# v2.10.0 更新日志

**发布日期**: 2026-05-22

## 🐛 Bug 修复

### 学习社区模块
- **修复新增问题后无法显示的问题**
  - 问题原因：前后端数据格式不匹配，后端返回的数据包装在 `{ code, message, data, timestamp }` 结构中，前端直接取 `res.data` 导致获取到的是整个响应对象而非数据数组
  - 修复方案：
    - 后端统一使用 Response 工具类返回数据，确保所有接口返回格式一致
    - 前端修正数据提取逻辑，从 `res.data.data` 获取实际数据
  - 影响接口：
    - `GET /api/community/questions` - 获取问题列表
    - `POST /api/community/questions` - 发布问题
    - `GET /api/community/favorites` - 获取收藏列表
    - `GET /api/community/questions/:id/answers` - 获取回答列表
  - 修复文件：
    - `smart-education-backend/routes/community.js`
    - `frontend/src/views/StudyCommunity.vue`

## 🔧 优化改进

### 后端接口规范化
- **统一学习社区所有接口响应格式**
  - 引入 Response 工具类，替换原有的手动构造响应对象
  - 所有接口返回统一格式：`{ code, message, data, timestamp }`
  - 使用语义化的响应方法：
    - `Response.success()` - 成功响应
    - `Response.error()` - 错误响应
    - `Response.badRequest()` - 参数错误
    - `Response.notFound()` - 资源不存在
    - `Response.forbidden()` - 禁止访问
  - 优势：
    - 提高代码可维护性
    - 统一错误处理
    - 自动记录日志
    - 自动添加时间戳

## 📝 技术细节

### 响应格式对比

**修复前**：
```javascript
// 不统一的响应格式
res.json({ success: true, data: [...] });
res.status(500).json({ success: false, message: '错误', error: error.message });
```

**修复后**：
```javascript
// 统一的响应格式
Response.success(res, data, '操作成功');
Response.error(res, '操作失败', 500);
```

### 前端数据提取

**修复前**：
```javascript
questions.value = res.data || [];  // 错误：获取到整个响应对象
```

**修复后**：
```javascript
questions.value = res.data.data || [];  // 正确：获取实际数据
```
