## 🔧 技术改进

### 后端优化

#### 新增流式接口
- 新增 `sendMessageStream` 控制器函数
- 支持Server-Sent Events (SSE)流式响应
- 使用 `ai.chatStream()` 方法调用AI流式API
- 实时推送AI回复片段到前端

#### AI自动生成对话标题
- 使用AI识别用户意图，生成简洁的对话标题
- 限制标题长度不超过15个字
- 自动移除引号等特殊字符
- 失败时自动降级到默认方式
- 同时支持普通传输和流式传输

**实现逻辑**:
```javascript
// 自动生成话题标题（如果是第一条消息）
if (topic.messages.length === 2 && topic.title === '新对话') {
  try {
    // 使用AI生成简洁的对话标题
    const titlePrompt = `请根据以下用户问题，生成一个简洁的对话标题（不超过15个字，不要加引号）：\n\n${message}`;
    const generatedTitle = await ai.chat(titlePrompt, []);
    topic.title = generatedTitle.trim().replace(/["""]/g, '').substring(0, 30);
  } catch (error) {
    // 如果AI生成失败，使用默认方式
    logger.warn('AI生成标题失败，使用默认方式:', error);
    topic.title = message.substring(0, 30) + (message.length > 30 ? '...' : '');
  }
}
```

#### 流式传输流程
```
接收用户消息
    ↓
保存用户消息到数据库
    ↓
检索知识库内容
    ↓
构建对话历史
    ↓
调用AI流式API
    ↓
逐块接收AI回复
    ↓
实时推送到前端（SSE）
    ↓
保存完整AI回复到数据库
    ↓
发送完成信号
```

#### SSE响应头配置
```javascript
res.setHeader('Content-Type', 'text/event-stream');
res.setHeader('Cache-Control', 'no-cache');
res.setHeader('Connection', 'keep-alive');
res.setHeader('X-Accel-Buffering', 'no'); // 禁用Nginx缓冲
```

#### 流式数据格式
```javascript
// 数据块
data: {"type":"chunk","content":"你"}

// 完成信号
data: {"type":"done","userMessage":{...},"assistantMessage":{...},"topic":{...}}

// 错误信号
data: {"type":"error","message":"错误信息"}
```

## 🔧 技术细节

### 流式传输实现

**后端 (Node.js + Express)**:
```javascript
// 设置SSE响应头
res.setHeader('Content-Type', 'text/event-stream');

// 调用AI流式API
await ai.chatStream(message, history, (chunk) => {
  // 实时推送数据块
  res.write(`data: ${JSON.stringify({ type: 'chunk', content: chunk })}\n\n`);
});

// 发送完成信号
res.write(`data: ${JSON.stringify({ type: 'done', ... })}\n\n`);
res.end();
```

**前端 (Vue 3 + Fetch API)**:
```javascript
// 发送请求
const response = await fetch(url, { method: 'POST', body: ... });

// 读取流式响应
const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  // 解码并处理数据
  const text = decoder.decode(value);
  // 更新UI
  messages.value[index].content += data.content;
}
```

## 🐛 Bug 修复

### 1. 左侧面板标题显示问题
- **问题**: 切换到对话标签页时，标题仍显示"助手管理"
- **修复**: 使用动态绑定，根据 `activeTab` 显示对应标题
- **代码**: `{{ activeTab === 'assistants' ? '助手管理' : '对话管理' }}`

### 2. 双层AI头像显示问题
- **问题描述**: 发送消息后，AI回复时会显示两个AI头像（一个是消息头像，一个是加载动画头像）
- **问题原因**: 流式传输时创建了AI消息占位符，同时 `isLoading` 状态也会显示独立的加载动画，导致重复显示
- **修复方案**: 移除独立的加载动画组件，将加载动画集成到AI消息占位符中

### 3. 对话自动命名优化
- **问题描述**: 新建对话后，对话名称固定显示为"新对话"或用户消息的前30个字符
- **优化方案**: 使用AI自动识别用户首次消息内容，生成简洁、语义化的对话标题
- **用户体验**: 对话标题更加准确、易于识别和管理

#### AI头像显示优化

**修复前的问题**:
```
用户: 你好
    ↓
[AI头像 + 加载动画]  ← 独立的加载组件
[AI头像 + 空消息]    ← 消息占位符
    ↓ 出现两个AI头像 ❌
```

**修复后的效果**:
```
用户: 你好
    ↓
[AI头像 + 加载动画]  ← 消息占位符中的加载动画
    ↓ 开始接收内容
[AI头像 + "你"]
[AI头像 + "你好"]
    ↓ 只有一个AI头像 ✅
```

#### 对话标题自动生成

## 📝 使用说明

### 流式对话体验
1. 在输入框中输入消息
2. 点击"发送"按钮
3. **立即看到自己的消息**
4. **看到AI逐字回复**（新功能）
5. AI回复完成后自动保存

### 流式传输优势
- **即时反馈**: 不用等待完整回复，立即看到AI开始回答
- **自然体验**: 像真人对话一样，逐字显示
- **减少焦虑**: 用户知道AI正在回复，不会感到等待
- **提前阅读**: 可以边看AI回复边思考，提高效率

### 对话管理
- 点击"助手"标签 → 标题显示"助手管理"
- 点击"对话"标签 → 标题显示"对话管理"

### 对话自动命名
1. 创建新对话
2. 发送第一条消息
3. **AI自动生成简洁的对话标题**（新功能）
4. 对话列表中显示生成的标题

### 标题生成规则
- 自动识别用户意图
- 生成不超过15个字的简洁标题
- 移除引号等特殊字符
- 如果AI生成失败，使用用户消息的前30个字符

### 手动修改标题
- 对话标题可以随时手动修改
- 右键点击对话 → 选择"重命名"
- 输入新标题并保存
