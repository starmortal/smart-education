### 需求 #2: 新增 AI 助手对话页面（登录后首页）
**描述**: 
- 创建一个新的 AI 助手对话页面，作为用户登录成功后的首页
- 页面采用左右分栏布局：左侧助手列表，右侧对话区域
- 实现助手管理和话题管理功能
- 参考 Cherry Studio 的对话界面逻辑

**涉及模块**: 
- [x] 前端
- [x] 后端
- [x] 数据库
- [ ] 其他

**优先级**: 
- [x] 高
- [ ] 中
- [ ] 低

**详细说明**:
```
1. 页面布局（参考 Cherry Studio）
   左侧区域（约 250-300px）：
   - 顶部：新建助手按钮
   - 助手列表：
     * 显示所有助手（可折叠/展开）
     * 每个助手下显示其话题列表
     * 支持助手的增删改查
     * 支持话题的增删改查
   - 底部：设置按钮（可选）

   右侧区域（剩余空间）：
   - 顶部工具栏：
     * 当前助手名称
     * 新建话题按钮
     * 助手设置按钮
     * 其他功能按钮（上传图片、网络搜索、知识库等）
   - 对话区域：
     * 显示当前话题的对话历史
     * 支持文本、图片、代码等多种消息类型
     * 消息渲染（支持 Markdown、代码高亮等）
   - 底部输入框：
     * 多行文本输入
     * 发送按钮
     * 附加功能按钮（上传、表情等）

2. 核心功能逻辑
   助手（Assistant）：
   - 助手是话题的父集
   - 每个助手可以设置：
     * 名称
     * 提示词（prompt）
     * 默认模型
     * 参数设置（温度、top-p、max tokens 等）
   - 单个助手下可以创建多个话题
   - 所有话题共用助手的参数设置和提示词

   话题（Topic）：
   - 话题是具体的对话会话
   - 每个话题包含：
     * 话题标题（可自动生成或手动设置）
     * 对话历史记录
     * 创建时间、更新时间
   - 话题归属于某个助手
   - 支持话题的切换、删除、重命名

   对话功能：
   - 发送消息给 AI
   - 接收 AI 回复（支持流式输出）
   - 消息历史记录
   - 上下文管理
   - 支持重新生成、编辑消息等

3. 数据结构设计
   Assistant 模型：
   {
     _id: ObjectId,
     userId: ObjectId,          // 所属用户
     name: String,              // 助手名称
     prompt: String,            // 提示词
     model: String,             // 默认模型
     temperature: Number,       // 温度参数
     topP: Number,              // top-p 参数
     maxTokens: Number,         // 最大 token 数
     createdAt: Date,
     updatedAt: Date
   }

   Topic 模型：
   {
     _id: ObjectId,
     assistantId: ObjectId,     // 所属助手
     userId: ObjectId,          // 所属用户
     title: String,             // 话题标题
     messages: [{               // 对话消息列表
       role: String,            // 'user' | 'assistant'
       content: String,         // 消息内容
       timestamp: Date
     }],
     createdAt: Date,
     updatedAt: Date
   }

4. 路由和导航
   - 新页面路由：/ai-chat 或 /assistant
   - 登录成功后默认跳转到此页面
   - 左侧导航栏添加对应图标（聊天图标）
   - 图标位置建议放在导航栏顶部或显著位置
```

**预期效果**:
```
1. 用户登录后进入 AI 助手对话页面
2. 左侧可以管理多个助手和话题
3. 点击不同的话题，右侧显示对应的对话内容
4. 可以在右侧输入框发送消息，AI 实时回复
5. 助手和话题的数据持久化保存
6. 界面简洁美观，交互流畅
7. 支持 Markdown 渲染、代码高亮等
```

**备注**:
```
技术实现建议：
- 前端组件：
  * 创建 frontend/src/views/AIChat.vue（主页面）
  * 创建 frontend/src/components/AssistantList.vue（助手列表）
  * 创建 frontend/src/components/ChatArea.vue（对话区域）
  * 使用 Element Plus 的 el-container、el-aside、el-main 等布局组件
  * 使用 marked 库渲染 Markdown
  * 考虑使用 highlight.js 进行代码高亮

- 后端接口：
  * POST /api/assistants - 创建助手
  * GET /api/assistants - 获取助手列表
  * PUT /api/assistants/:id - 更新助手
  * DELETE /api/assistants/:id - 删除助手
  * POST /api/topics - 创建话题
  * GET /api/topics/:assistantId - 获取助手下的话题列表
  * PUT /api/topics/:id - 更新话题
  * DELETE /api/topics/:id - 删除话题
  * POST /api/chat - 发送消息并获取 AI 回复
  * GET /api/messages/:topicId - 获取话题的消息历史

- 数据库：
  * 创建 Assistant 模型（smart-education-backend/models/Assistant.js）
  * 创建 Topic 模型（smart-education-backend/models/Topic.js）
  * 创建 Message 模型或将消息嵌入 Topic 中

- AI 集成：
  * 复用现有的 OpenAI 集成（smart-education-backend/utils/ai.js）
  * 支持流式输出
  * 实现上下文管理

参考资源：
- Cherry Studio 文档：https://docs.cherry-ai.com/cherry-studio/preview/chat
- 已提供的界面截图

优先级说明：
- 此功能为核心功能，优先级高
- 建议先实现基础的助手和话题管理
- 再逐步完善对话功能和高级特性
```
